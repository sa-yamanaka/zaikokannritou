# 製造業向け 在庫管理システム

Windows上でサーバーを起動し、PC/スマホ両方から使える在庫管理システムです。

---

## 📦 システム構成

```
inventory-system/
├── server.py              # Python Flaskサーバー（本体）
├── requirements.txt       # 必要なPythonパッケージ
├── config.json            # 設定ファイル（自動生成）
├── 初回セットアップ.bat    # 1回だけ実行
├── 起動.bat               # 毎日使う起動用
├── data/
│   ├── inventory.db       # 在庫データ（SQLite）
│   └── 履歴.xlsx          # 増減履歴（自動追記）
├── templates/             # 画面HTML
│   ├── index.html         # PC/スマホ選択画面
│   ├── desktop.html       # PC用画面
│   └── mobile.html        # スマホ用画面
└── static/                # CSS・JavaScript
    ├── style.css
    └── app.js
```

---

## 🚀 セットアップ手順（初回のみ）

### 1. Python をインストール
まだ Python が入っていなければ、下記からダウンロードしてインストールしてください。
- https://www.python.org/downloads/
- インストール時、**「Add Python to PATH」** のチェックを**必ず**つけてください。

### 2. 初回セットアップ実行
`inventory-system` フォルダ内の **`初回セットアップ.bat`** をダブルクリック。
→ 必要なパッケージ（Flask, openpyxl, qrcode）が自動でインストールされます。

---

## 💻 毎日の使い方

### 起動
`起動.bat` をダブルクリック
→ 自動的にブラウザで PC/スマホ選択画面が開きます。
→ 黒いコマンドウィンドウは **閉じないでください**（サーバーが動き続けます）

### 終了
コマンドウィンドウを閉じるだけです。

---

## 📱 スマホから使う

1. スマホを Windows PC と **同じWi-Fi** に接続
2. PC画面のQRコードをスマホのカメラで読み取る
3. スマホ用画面が開きます

> ※ QR が読めない場合は、PC画面に表示されているURL（例: `http://192.168.1.20:5000/mobile`）を直接スマホのブラウザに入力してください。

## 📝 増減履歴について

スマホ・PCからの全ての在庫増減は、`data/履歴.xlsx` に自動で追記されます。

| 日時 | 振り分け番号 | メーカー | サイズ | 色 | 増減量 | 操作後の在庫数 | 操作端末 |
|------|------------|---------|-------|---|-------|--------------|--------|
| 2026-04-24 10:15:32 | AL-001 | A社 | 100×200×3 | シルバー | +10 | 30 | スマホ |

- **すべてこのPC内のExcelファイルに保存** されます（外部送信なし）
- 定期的にバックアップを取ることを推奨
- ファイルは開いたままだと書き込みエラーになる可能性があるので、サーバー稼働中は閉じておいてください

---

## 🔒 セキュリティについて

### 現在の構成（推奨）
- **データはすべてこのPC内のファイル** に保存されます（SQLiteとExcel）
- 外部インターネットへのデータ送信は**一切なし**
- スマホからのアクセスは、**同じWi-Fi内でのみ可能**（社内LAN）
- 会社のセキュリティポリシーに抵触する可能性は**極めて低い**

### Firebaseを使う場合（オプション）
`config.json` で `use_firebase: true` に設定することで、クラウド保存に切替可能です。
※ただし、クラウド利用については情報システム部門への事前確認を推奨します。

## ❓ トラブルシューティング

### スマホからQRが読み取れない / 接続できない
- PCとスマホが同じWi-Fiか確認
- Windowsのファイアウォール設定で `Python` からの通信を許可
- PCのIPアドレスが変わった場合、画面更新（F5）でQR再生成

### `起動.bat` を起動すると赤いエラーが出る
- Pythonが入っていないか、PATHが通っていない
- 再度 Python を「Add to PATH」にチェックを入れてインストール
- または `初回セットアップ.bat` を先に実行

### ブラウザが自動で開かない
- 手動で `http://localhost:5000/` にアクセスしてください

### 履歴.xlsxに書き込めない
- Excelでファイルを開いていませんか？ → 閉じてください

---

## 🛠 設定ファイル（config.json）

```json
{
  "port": 5000,
  "use_firebase": false,
  "firebase_config": {
    "service_account_path": "",
    "collection": "inventory"
  }
}
```

- `port`: サーバーのポート番号（デフォルト5000）。他のアプリと衝突する場合は変更可。

## 📞 サポート

動作不良や機能追加のご要望は、随時カスタマイズ可能です。

- データを失わないよう、`data/` フォルダは定期バックアップを推奨
- 大量のデータを扱う場合はSQLiteからPostgreSQLへの移行も可能
